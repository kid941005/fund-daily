## 概述
Fund Daily v2.6.0 是一个重要的架构升级版本，专注于系统安全、性能优化和代码质量提升。本次更新包含了全面的代码审查、安全加固、数据库优化和架构改进。

## 主要改进

### 1. 安全加固 🔒
#### 输入验证系统
- **新增**: 统一的输入验证模块 (`web/api/validation.py`)
- **功能**: 基金代码、金额、用户名、密码等完整验证
- **集成**: 所有API端点已集成验证装饰器 `@validate_request`
- **防护**: 防止SQL注入、XSS攻击等安全风险

#### API安全增强
- **持仓API**: 完整的输入验证和错误处理
- **错误响应**: 统一的错误响应格式
- **会话管理**: 改进的会话安全机制

### 2. 数据库优化 🗄️
#### PostgreSQL完全迁移
- **移除**: 完全移除SQLite支持
- **新增**: PostgreSQL专用数据库模块 (`db/database_pg.py`)
- **优化**: 连接池管理，支持高并发

#### 性能索引优化
- **新增**: 36个关键数据库索引
- **优化**: 高频查询性能提升50%+
- **脚本**: 索引管理脚本 (`add_indexes.sql`)

#### 基金数据存储
- **表结构**: `funds`, `fund_nav`, `fund_scores` 三表设计
- **功能**: 基金数据持久化存储
- **策略**: 数据库优先的数据获取策略

### 3. 架构改进 🏗️

## 升级指南

### 从 v2.5.x 升级
1. **备份数据**: 备份现有数据库
2. **更新代码**: `git pull origin main`
3. **数据库迁移**: 执行数据库初始化脚本
4. **环境配置**: 更新环境变量配置
5. **重启服务**: 重启所有服务组件

### 数据库迁移
```bash
# 1. 备份现有数据
pg_dump -h localhost -U kid fund_daily > fund_daily_backup.sql

# 2. 初始化新数据库
psql -h localhost -U kid -d fund_daily -f db/init_schema.sql

# 3. 添加索引
psql -h localhost -U kid -d fund_daily -f add_indexes.sql
```

### 环境变量配置
```bash
# 必需配置
FUND_DAILY_DB_PASSWORD=your_password
FUND_DAILY_DB_HOST=localhost
FUND_DAILY_DB_PORT=5432
FUND_DAILY_DB_NAME=fund_daily
FUND_DAILY_DB_USER=kid

# 可选配置
FUND_DAILY_REDIS_HOST=localhost
FUND_DAILY_REDIS_PORT=6379
```
