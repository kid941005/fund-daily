# 环境变量迁移指南

## 概述

为了统一环境变量命名规范，项目已将部分环境变量添加 `FUND_DAILY_` 前缀。系统同时支持新旧两种变量名，但**推荐使用新变量名**。

## 迁移时间线

- **v2.6.8+**: 支持新旧两种变量名
- **v3.0.0**: 计划移除旧变量名支持

## 变量名变化对照表

## 优先级规则

系统按以下优先级读取环境变量：

1. **新变量名** (`FUND_DAILY_*`)
2. **旧变量名** (无前缀)
3. **默认值**

示例：
```python
# 如果同时设置了新旧变量，新变量优先
FUND_DAILY_REDIS_HOST=redis-new  # 这个会被使用
REDIS_HOST=redis-old             # 这个被忽略

# 如果只设置了旧变量，旧变量会被使用
REDIS_HOST=redis-old             # 这个会被使用（因为没有新变量）
```

## 迁移步骤

### 1. 更新 `.env` 文件

将现有 `.env` 文件中的变量名更新：

```bash
# 迁移前
REDIS_HOST=localhost
CORS_ORIGINS=*
LOG_LEVEL=INFO
DEBUG=false

# 迁移后
FUND_DAILY_REDIS_HOST=localhost
FUND_DAILY_CORS_ORIGINS=*
FUND_DAILY_LOG_LEVEL=INFO
FUND_DAILY_DEBUG=false
```

### 2. 更新部署配置

更新 Docker Compose、Kubernetes、CI/CD 等部署配置中的环境变量。

### 3. 更新文档

更新项目文档、Wiki 等中的环境变量说明。

## 向后兼容性

当前版本 (v2.6.8+) 完全向后兼容，支持新旧两种变量名。但建议尽快迁移到新变量名。

## 故障排除

### 问题：配置未生效

**可能原因**：
1. 环境变量名拼写错误
2. `.env` 文件未正确加载
3. 应用未重启

**解决方案**：
1. 检查变量名拼写
2. 确认 `.env` 文件路径正确
3. 重启应用服务

### 问题：新旧变量冲突

**解决方案**：
1. 统一使用新变量名
2. 删除旧变量名定义
3. 或确保新变量名优先级更高
