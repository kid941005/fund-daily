**问题**：API端点缺乏输入验证，存在安全风险。

**解决方案**：
1. 创建统一的输入验证模块
2. 为所有API端点添加参数验证
3. 实现数据清洗和转义

**具体任务**：
- [ ] 创建 `web/api/validation.py` 模块
- [ ] 为 `/api/holdings` 端点添加输入验证
- [ ] 为 `/api/funds` 端点添加输入验证
- [ ] 为 `/api/quant/*` 端点添加输入验证

**验证规则示例**：
```python
def validate_fund_code(code: str) -> bool:
    """验证基金代码格式"""
    if not code or not isinstance(code, str):
        return False
    # 基金代码通常为6位数字
    return code.isdigit() and len(code) == 6

def validate_amount(amount: float) -> bool:
    """验证金额格式"""
    if not isinstance(amount, (int, float)):
        return False
    return amount >= 0 and amount <= 1000000  # 最大100万
```

**风险**：重构可能引入新bug
**缓解**：

- 充分测试，确保功能不变
- 保留回滚方案

### 5.2 业务风险
**风险**：影响用户体验

### 5.3 时间风险

**缓解**：
