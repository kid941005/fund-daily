## 🔴 P0 问题（紧急，必须修复）

### P0-1: Dockerfile 中残留 Flask 配置
- **位置**: `Dockerfile`
- **问题**: 第 54 行 `FLASK_APP=web/app.py` — 项目已迁移到 FastAPI，`web/app.py` 不存在
- **影响**: Docker 构建虽使用 `CMD ["uvicorn", "web.api_fastapi.main:app", ...]`（正确），但 `FLASK_APP` 环境变量是遗留错误，可能导致其他脚本引用失败
- **建议**: 删除 `FLASK_APP=web/app.py` 这一行

### P0-2: start.sh 引用不存在的 Flask 入口
- **位置**: `start.sh`
- **问题**: `exec python3 web/app.py` — `web/app.py` 不存在
- **影响**: 直接执行 start.sh 会报错
- **建议**: 改为 `exec uvicorn web.api_fastapi.main:app --host 0.0.0.0 --port $FUND_DAILY_SERVER_PORT`

### P0-3: web/requirements.txt 包含 Flask 残留依赖
- **位置**: `web/requirements.txt`
- **问题**: 包含 `flask==3.0.0` 和 `flask-cors==4.0.0`
- **影响**: 项目已完全迁移到 FastAPI，这些依赖无人使用，已废弃
- **建议**: 删除 Flask 相关行，或确认是否被 `web/` 目录下的脚本使用

### P0-4: CORS 配置允许所有来源（生产安全风险）
- **位置**: `web/api_fastapi/main.py` 第 65 行
- **问题**: `allow_origins=["*"]` 允许所有来源访问 API
- **影响**: 生产环境中恶意网站可调用用户 API，存在认证令牌被盗风险
- **建议**: 改为明确的白名单，如 `["https://fund.example.com"]` 或通过环境变量配置

### P0-5: .env 文件包含明文密码且可能已提交
- **位置**: `.env`
- **问题**: 
  - `DB_PASSWORD=941005`（明文）
  - `FUND_DAILY_DB_PASSWORD=941005`（明文重复）
  - `.gitignore` 中 `.env` 可能未被正确排除
- **影响**: 数据库密码泄露
- **建议**: 
  1. 确认 `.gitignore` 包含 `.env`
  2. 创建 `.env.example` 模板（不包含真实密码）
  3. 考虑使用 Docker secrets 或 Vault 管理密码

### P0-6: docker-compose.yml 中密码默认值不安全
- **位置**: `docker-compose.yml` 和 `docker-compose.fastapi.yml`
- **问题**: 
  - `POSTGRES_PASSWORD: ${POSTGRES_PASSWORD:-941005}`（默认密码）
  - `FUND_DAILY_DB_PASSWORD:${POSTGRES_PASSWORD:-941005}`（默认密码）
- **影响**: 部署时若未设置环境变量，使用默认密码
- **建议**: 默认值改为空字符串或随机生成，强制要求配置密码

### P0-7: 测试失败 - jwt_auth 导出缺失
- **位置**: `tests/test_jwt_auth.py`
- **问题**: 测试文件导入了不存在的符号：
  - `get_token_from_header`
  - `jwt_required`
  - `jwt_optional`
- **影响**: 6 个 JWT 测试全部失败（ImportError）
- **建议**: 
  - 如果这些函数应该存在：在 `src/jwt_auth.py` 中实现
  - 如果不再需要：更新测试文件移除这些导入

## 🟡 P1 问题（重要，应修复）

### P1-1: 数据库层双轨并行（同步 vs 异步）
- **位置**: `db/` 目录（同步）vs `src/db/` 目录（异步）
- **问题**: 项目同时维护了两套数据库操作层：
  - 同步层：`db/database_pg.py`（旧）
  - 异步层：`src/db/async_database.py`、`src/db/async_crud.py`、`src/db/repositories.py`（新）
  - 所有 FastAPI 路由仍在使用同步层 `db import database_pg as db`
- **影响**: 异步层未被实际使用，代码冗余，维护成本高
- **建议**: 
  1. 统一迁移到异步层
  2. 或删除异步层避免混淆

### P1-2: .env.example 仍有 Flask 注释残留
- **位置**: `.env.example`
- **问题**: 注释中包含 `# Flask Secret Key` 和 `# 生成命令: python -c "import secrets...`
- **影响**: 文档不准确
- **建议**: 清理 Flask 相关注释，改为 `# 应用密钥`

### P1-3: docker-compose.yml.flask_backup 未清理
- **位置**: `docker-compose.yml.flask_backup`
- **问题**: Flask 版本的 docker-compose 配置残留
- **影响**: 可能被误用，造成部署混乱
- **建议**: 删除或移至 backup/ 目录

### P1-4: 同步数据库连接被导入到 FastAPI 异步路由中
- **位置**: `web/api_fastapi/routers/` 多个路由文件
- **问题**: 所有路由文件（funds.py, holdings.py, auth.py, analysis.py 等）使用 `from db import database_pg as db`（同步 psycopg2），但在 `async def` 路由中调用
- **影响**: 同步数据库调用会阻塞事件循环，降低并发性能
- **建议**: 迁移到 `src/db/async_database.py` 的异步层

### P1-5: 环境变量前缀不一致
- **位置**: 多处
- **问题**: 
  - `.env` 中有 `DB_PASSWORD`（无前缀）
  - 同时有 `FUND_DAILY_DB_PASSWORD`（有前缀）
  - `FUND_CODES`、`REPORT_TIME`、`ALERT_THRESHOLD` 等无前缀
- **影响**: 配置查找复杂，容易出错
- **建议**: 统一使用 `FUND_DAILY_*` 前缀，或在 `src/config.py` 中明确映射

### P1-6: 测试配置 - 异步 fixture 在同步测试中使用
- **位置**: `tests/test_async_db.py`
- **问题**: 测试使用 `@pytest.fixture` 的 `async_db` 但测试函数是同步的
- **影响**: 27 个 ERROR（pytest-asyncio 版本兼容问题）
- **建议**: 使用 `@pytest_asyncio.fixture` 或将测试标记为 `@pytest.mark.asyncio`

### P1-7: API 路由中的重复认证逻辑
- **位置**: `web/api_fastapi/routers/` 下多个文件
- **问题**: 每个路由文件都定义了 `_get_user_id()` 和 `_auth_required()` 重复函数
- **影响**: 代码重复，维护困难
- **建议**: 统一使用 `web/api_fastapi/deps.py` 中的 `get_current_user` 依赖

### P1-8: ConfigManager 在 jwt_auth.py 中被重复尝试导入
- **位置**: `src/jwt_auth.py`
- **问题**: `try/except` 块中尝试导入 `get_config`，失败后使用硬编码默认值
- **影响**: 如果配置有问题，可能静默使用不安全的默认 JWT secret
- **建议**: 启动时验证配置，无效配置应报错而非静默回退
